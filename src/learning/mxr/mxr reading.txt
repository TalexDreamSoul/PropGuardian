 一、Git

- 工作流程：
    - 本地工作流程：工作区代码添加到暂存区，暂存区代码提交到本地版本库。
    - 完整运行流程：本地版本库代码推送到远程仓库，从远程仓库拉取代码到本地。
- 安装 Git 客户端：[Git 官网](https://git-scm.com/)下载安装。
- 设置邮箱和用户名
    ```
    git config --global user.email "自己的邮箱"
    git config --global user.name "Your Name"
    ```
    查看设置是否成功：
    ```
    git config --list
    ```
    其他常用命令：
    - 创建文件夹测试，如`E:\gitTest`，进入该目录。
    - 初始化：`git init 项目名`。
    - 提交到暂存区：`git add 文件名`（指定文件）或`git add .`（所有文件）。
    - 查看暂存区：`git ls-files`。
    - 恢复暂存区文件到工作区：`git reset 文件名`（指定文件）或`git reset .`（所有文件）。
    - 提交到本地仓库：`git commit -m 注释`。
- IDEA 集成 Git：
    - 在 IDEA 设置中集成 Git。
    - 初始化本地 Git 仓库项目，选择需要添加到 Git 仓库的项目。
    - 安装 `.ignore` 插件，设置忽略文件功能，排除不需要提交的文件。
- 将代码添加到本地仓库：添加到暂存区，未提交到本地仓库。
- Git 本地库操作：
    - 查看历史版本记录。
    - 回退到之前的历史版本。
    - 将选中历史版本与工作区进行比对，了解修改和添加的代码。
    - Git 只将代码提交到本地仓库，需远程仓库实现多人代码共享。

二、Gitee
- Gitee：开源中国出品的云端软件开发协作平台，提供代码托管、项目管理、协作开发等功能，适用于个人、团队和企业。
- 账号注册及设置：
    - 注册账号，个人主页点击个人设置绑定邮箱。
    - 生成 SSH 公钥。
- IDEA 集成 Gitee：
    - IDEA 插件搜索 Gitee 并安装。
    - 添加 Gitee 账号，输入绑定的邮箱和密码。
    - 推送本地仓库代码到远程仓库：
        - 首次推送：按照提示操作。
        - 推送成功后，修改内容再次推送时，可右键修改的文件，选择`Git`->`Repository`->`Push`。若提交文件本地仓库没有，需先添加到本地仓库。
- 从远程仓库克隆代码**：在 Gitee 上选择要克隆的项目，按照提示操作。
- 仓库管理：
    - 在 Gitee 管理列表中，通过仓库成员管理加入小组成员，实现多人共同使用仓库。
    - 在仓库设置中，可设置开源和私有。
- 冲突解决：本地仓库向远端仓库提交代码时，可能出现文件冲突，多发生在公共配置文件中。可通过良好规范和沟通尽量避免冲突。提交时若冲突，会弹框提示，点击`merge`（合并）解决冲突。
